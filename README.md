# Nginx Log Security Monitoring Dashboard

## 概要

本プロジェクトはAWS EC2上でNginx環境を構築し、
Webアクセスログを解析・可視化するダッシュボードを作成したものです。

当初は単純なログ解析およびアクセス集計ツールとして設計していましたが、実際のログを分析する中で、特定IPからの大量404エラーやスキャン系リクエストの偏りを確認しました。

この観測をきっかけに、単なるアクセス解析ではなく「異常アクセス検知」の観点が必要だと判断し、攻撃パターン分類およびリスクスコアリング機能を追加しました。

結果として、本プロジェクトはWebアクセス解析に加え、簡易的なセキュリティ監視ダッシュボードへと発展しました。

---

## 目的

- Nginxアクセスログの構造を理解する  
- アクセス傾向とエラー発生状況を可視化する  
- 異常アクセス（スキャン・攻撃）の兆候を検知する  
- ログ解析を通じてサーバー運用の基礎を習得する  
- Pythonによるデータ処理と可視化の実践力を身につける  

---

## 環境

- クラウド: AWS EC2 (Ubuntu Server)  
  - 仮想サーバーを構築し、外部からアクセス可能なWeb環境を構築
- OS: Ubuntu Server 22.04  
- Webサーバー: Nginx（HTTP公開環境として使用）  
- 開発言語: Python 3.x  
- 使用ライブラリ: matplotlib, collections（defaultdict）  

---

## 実装内容

### 1. アクセスログ解析
- IPアドレスごとのアクセス数を集計
- HTTPステータスコード別の件数を集計

---

### 2. 攻撃パターン分類
アクセスURLをもとに以下を分類：

- phpunit系攻撃
- Laravel関連スキャン
- WordPress探索（wp-）
- admin / login などの管理画面探索
- CGI・xmlrpcなどの脆弱性探索
- その他（unknown）

---

### 3. リスクスコアリング
IPごとに以下のルールでスコア化：

- アクセスごと +1  
- 404エラー +2  
- スキャン系URL +5  
- phpunitアクセス +10  

→ 危険度の高いIPをランキング表示

---

### 4. 可視化機能
- matplotlibによるIPアクセス棒グラフ生成  
- HTMLダッシュボード出力  
- 分析結果のリアルタイム可視化  

---

## 学んだこと

- Nginxアクセスログの構造理解  
- HTTPステータスコードの意味（200 / 304 / 404など）  
- Linux環境でのログ解析手法  
- Pythonによるデータ集計（defaultdict）  
- ログから異常パターンを抽出する考え方  
- 可視化による情報整理の重要性  
- セキュリティ視点（攻撃スキャン・脆弱性探索）の基礎理解  

---

## 苦労したこと

- ログフォーマットの違いによる解析ミス  
- URL抽出位置のズレ（partsの扱い）  
- スコア設計が単純すぎて数値が暴れる問題  
- unknown分類が増えすぎて意味が薄くなる問題  
- Pythonのインデントエラー（Tabとスペース混在）
